## Description

### Problem

Nowdays it still is very common that we visit restaurants that control what their customers consumes via paper. Although simple, that approach brings some operational inneficiences like:

- How to track order from beginning to delivery?
- What if the order note gone?
- How to keep an orders history and extract metrics from it?

### Solution

WaiterAPP is a tech solution for that and many other problems. Restaurants can manage their menu, categorize products, keep orders history and more.

The app is divided in three parts:

- An idiomatic HTTP API to interact with data;
- A web backoffice with a simple UI to realize administrative tasks;
- A mobile app where waiters take and track orders in real time;

This repository contains all implementation of the API.
